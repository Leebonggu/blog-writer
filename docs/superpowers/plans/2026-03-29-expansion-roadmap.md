## 확장 방향

### Phase 2: 카테고리 확장

현재 템플릿 구조(`features/blog/templates/`)에 파일만 추가하면 됨.

| 카테고리 | 핵심 포인트 | 난이도 |
|---------|-----------|--------|
| 카페 | 분위기/디저트/음료 묘사, 작업하기 좋은지 | 템플릿 1개 |
| 숙소/호텔 | 체크인 경험, 룸 컨디션, 조식, 위치 | 템플릿 1개 |
| 병원/클리닉 | 시술 후기, 의사 상담, 가격, 경과 | 템플릿 1개 |
| 뷰티/네일 | 시술 과정, 디자인, 가격대, 만족도 | 템플릿 1개 |
| 헬스/PT | 시설, 트레이너, 프로그램, 가격 | 템플릿 1개 |

### Phase 3: 플랫폼 확장

블로그 외 다른 플랫폼용 글 생성.

| 플랫폼 | 차이점 | 변경 범위 |
|--------|--------|----------|
| 인스타그램 | 짧은 캡션, 해시태그, 이미지 중심 | 새 base 프롬프트 + 해시태그 생성 로직 |
| 네이버 플레이스 리뷰 | 300자 이내, 별점 기반, 간결 | 새 base 프롬프트 (짧은 버전) |
| 카카오맵 리뷰 | 플레이스 리뷰와 유사 | base 재활용 |
| 유튜브 스크립트 | 영상 대본, 타임스탬프 | 완전 새 base |
| 유튜브 숏츠 대본 | 60초 숏폼 대본 + 자막 | 새 base (아래 상세) |

구현 방식: `features/blog/` → `features/content/`로 리네이밍하고, platform 개념 추가.

```
features/content/
├── platforms/
│   ├── naver-blog/    ← 현재
│   ├── instagram/
│   ├── naver-place/
│   └── youtube/
├── templates/          ← 카테고리 (공유)
└── tone-presets.ts     ← 공유
```

### Phase 3-1: 유튜브 숏츠 대본 상세

이미지 비전 분석 + 가게 정보로 60초 숏츠 대본 생성.

**출력 형식:**
```
[0:00] 훅
  나레이션: "노원역에서 삼겹살 맛집 찾으세요?"
  자막: 노원역 삼겹살 맛집 🔥
  화면: 고기 클로즈업

[0:03] 가게 소개
  나레이션: "바로 여기, 목삼화로입니다"
  자막: 목삼화로 노원 본점
  화면: 외관 사진

[0:08~0:25] 메뉴 하이라이트 (메뉴별 반복)
  나레이션: "특목살 17,000원, 숙성 목살의 끝판왕"
  자막: 특목살 17,000원
  화면: 메뉴 사진

[0:25~0:45] 먹방/리액션
  나레이션: "한 입 베어무는 순간 육즙이..."
  자막: 육즙 폭발 🤤
  화면: 먹는 장면 or 클로즈업

[0:45~0:55] 가게 정보
  나레이션: "노원역 1번 출구 도보 3분"
  자막: 📍노원역 1번 출구 3분 / ⏰매일 17:30-02:00
  화면: 지도 or 외관

[0:55~0:60] CTA
  나레이션: "저장해두세요!"
  자막: 저장 📌 + 팔로우
  화면: 로고/엔딩
```

**구현:**
- 새 platform: `features/content/platforms/youtube-shorts/`
- base 프롬프트: 타임스탬프 + 나레이션/자막/화면 3트랙 구조
- 카테고리 템플릿 재활용 (맛집/배달/소품샵 글 구조를 숏츠 구조로 변환)
- 이미지 비전 분석 활용: 사진 보고 화면 지시 + 나레이션 맞춤 생성
- 출력 UI: 타임라인 뷰 + 나레이션/자막 각각 복사 버튼
- 자막 파일(.srt) 다운로드 기능 추가 가능

### Phase 4: 사용자 기능

| 기능 | 설명 | 필요 인프라 |
|------|------|-----------|
| 생성 히스토리 | 최근 생성 기록 저장 | localStorage (DB 없이) |
| 레퍼런스 글 저장 | 자주 쓰는 톤 저장 | localStorage |
| 사용자 인증 | 로그인/회원가입 | NextAuth + DB |
| API 키 자체 입력 | 사용자가 본인 키 사용 | 폼 필드 추가 |
| 과금 | 토큰 사용량 기반 과금 | Stripe + DB |

### Phase 5: 자동 포스팅

| 기능 | 방식 | 리스크 |
|------|------|--------|
| 네이버 블로그 자동 발행 | Selenium/Playwright로 로그인 → 에디터 조작 | 네이버 차단 가능 |
| 이미지 자동 업로드 | 에디터에 이미지 삽입 자동화 | 위와 동일 |

→ 리스크 높아서 후순위. "복붙" 방식이 안전함.
